# Capturing Terminal QR Codes to MP4

This document describes how to capture animated QR codes generated by `qr_encoder_lite.py` in a terminal window and save them to an MP4 file, which can later be decoded by `qr_decoder_lite.py`.

## Method: Screen Capture with FFmpeg (`x11grab`)

The most reliable method to capture block-character QR codes exactly as they appear in the terminal is to perform a direct screen capture of the X11 window using `ffmpeg`.

### Prerequisites

1.  **A graphical environment (X11)**: You need an X server running. If you are in a headless environment, you can use `Xvfb`.
2.  **FFmpeg**: Installed with `x11grab` support (`sudo apt-get install ffmpeg`).
3.  **A terminal emulator**: Such as `xterm`, `gnome-terminal`, or `alacritty`. (For `xterm`, ensure you have proper monospace fonts installed like `xfonts-base`).

### Steps to Capture

#### 1. Start the Terminal

Open your terminal emulator. Make sure the font size is appropriate and the window is large enough so the QR codes are not truncated.
If you're using `xterm`:
```bash
xterm -geometry 80x50 -fa 'Monospace' -fs 10 &
```

#### 2. Find the Display or Window ID

If you are capturing the entire display (e.g., `:0.0`), you can just use that.
Alternatively, you can capture a specific window if you know its position and size, or run a dedicated virtual frame buffer (`Xvfb`) just for the capture.

#### 3. Start FFmpeg Recording

Use `ffmpeg` to start recording the screen. Adjust the `-video_size`, `-framerate`, and display `-i` parameters as needed.

```bash
ffmpeg -video_size 1024x768 -framerate 10 -f x11grab -i :0.0 -c:v libx264 -pix_fmt yuv420p output.mp4
```

*   `-framerate 10`: 10 FPS is usually sufficient for QR code streams, but you can increase it if your `qr_encoder_lite.py` delay (`-d`) is very small.
*   `-f x11grab -i :0.0`: Captures the primary X11 display.
*   `-c:v libx264 -pix_fmt yuv420p`: Ensures the resulting MP4 is highly compatible.

#### 4. Run the Encoder

While FFmpeg is recording, run your `qr_encoder_lite.py` command inside the target terminal. For example:

```bash
cat my_data.txt | base64 | python3 qr_encoder_lite.py -n 50 -d 0.5
```

#### 5. Stop Recording

Once the script finishes displaying all QR chunks, stop the `ffmpeg` process (press `Ctrl+C` in the terminal where `ffmpeg` is running).

### Decoding the Captured MP4

You can then pass the resulting `output.mp4` directly to the decoder:

```bash
python3 qr_decoder_lite.py output.mp4 > decoded.txt
```

### Headless Automation Example

If you need to automate this process on a headless server, you can use `Xvfb` and `xterm` together:

```bash
#!/bin/bash
# Start a virtual X server
export DISPLAY=:99
Xvfb :99 -screen 0 1024x768x24 > /dev/null 2>&1 &
XVFB_PID=$!
sleep 2

# Start a terminal running the QR encoder
xterm -geometry 80x50+0+0 -fa 'Monospace' -fs 10 -e "echo 'Hello World' | base64 | python3 qr_encoder_lite.py -n 50 -d 0.5; sleep 5" &
XTERM_PID=$!
sleep 1

# Capture with ffmpeg for a fixed duration (e.g., 10 seconds)
ffmpeg -video_size 1024x768 -framerate 10 -f x11grab -i :99.0 -t 10 -c:v libx264 -pix_fmt yuv420p test_capture.mp4 -y

# Cleanup
kill $XTERM_PID
kill $XVFB_PID
```
